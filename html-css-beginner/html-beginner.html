Html och Css introduktion: Html - Hello World

#

Man brukar säga att HTML ett markeringsspråk och inte ett programmerings-språk. Det betyder att det inte går att skapa logik i HTML. Det går inte att skapa variabler, funktioner eller loopar i HTML. Alltså, html kan inte "utföra något" själv utan är bara ett sätt för att visa webblsären hur den ska visa upp innehållet.

Jag kommer att på ett grundläggande sätt att gå igenom en del av en webbsidas struktur. Vi kommer att gå igenom hur vi skapar en webbsida och hur vi kan använda oss av html för att skapa en webbsida. Vi kommer också att gå igenom hur vi kan använda oss av css för att styla vår webbsida. Vi börjar med html.

Innan vi börjar

#

En helt vanlig *.html-fil kan köras lokalt, direkt ur sin mapp på datorn. Alltså du kan gå in i mappen den ligger i och klicka på den så kommer den att öppnas i webbläsaren och se ut som den ska, utan att ha några tillägg eller lägga den på en webbserver.

Men vi kommer att använda oss av en webbserver för att köra våra sidor. Inte för att det krävs, utan för att det är behändigt då webbsidan uppdateras när du trycker på spara och du ser direkt hur din ändring ser ut.

Vi kommer att använda Visual Studio Code till alla exampel och jag tar förgivet att du använder det eftersom det finns några funktioner som fungerar annorlunda i andra IDEs.

Liveserver

Vi kommer att använda oss av en webbliveserver som heter Live Server. Det är en tilläggsfunktion till Visual Studio Code som gör att vi kan köra vår webbsida lokalt. Det är enkelt att installera och använda. Du kan installera det via Extensions i Visual Studio Code.

Struktur

#

I html använder vi alltid samma struktur när vi skapar en sida. Vi börjar med att skriva ! och sedan trycker vi på tab. Detta kommer att skapa en mall för oss som vi kan fylla i. Det kommer att se ut såhär: 

<!DOCTYPE html>
<html lang="sv">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>

</body>
</html>

DOCTYPE-taggen

Det här är grundskalet i alla HTML5-filer. Gamla html-versioner som till exempel HTML4 så fanns det inget krav för tex DOCTYPE. Så ibland när man kikar på html-koder saknas dessa och kan se lite olika ut mot vår kod. DOCTYPE är en deklaration som säger vilken version av html som används. Detta är viktigt för webbläsaren att veta så att den kan visa sidan korrekt.

html-taggen

html är själva grunden för vår webbsida. Allt som vi skriver i vår webbsida ska ligga inom html. Det är också här vi anger vilket språk som används på sidan. Detta är viktigt för webbläsaren att veta så att den kan visa sidan korrekt.

Värt att notera är också att de flesta webbläsare förstår att det är html som används även om det saknas DOCTYPE och därför kan vi egentligen bara skriva ut html helt slumpmässigt i dokumentet. Men vi undviket detta.

head-taggen

head är där vi lägger all information som INTE ska visas i själva webbsidan. Det kan vara information som webbläsaren behöver för att visa sidan korrekt. Det kan också vara information som sökmotorer behöver för att kunna indexera sidan. Det kan också vara information som vi vill att webbläsaren ska använda för att visa sidan korrekt. Det kan vara information som vi vill att webbläsaren ska använda för att visa sidan korrekt.

title-taggen

En av få synliga saker som vi kan lägga i head är title. Detta är den titel som kommer att visas i webbläsarens flik. Det är också den titel som kommer att visas när sidan dyker upp i sökmotorer. På äldre datorer fanns det i stort sett alltid en titel-rad i alla fönster på datorn. Detta är den titel som kommer att visas i webbläsarens flik. Där av namnet title. Namnet skall inte förväxlas med h1 som är en rubrik.

body-taggen

body är där vi lägger all information som ska visas i själva webbsidan. Det är här vi lägger allt innehåll som vi vill ska visas i webbläsaren, men även där kan vi lägga innehåll som inte syns för användaren. Vi kommer dit sen.

Kommentarer och upplägg

#

Jag kommer inte att skriva ut den kompletta koden för varje exempel. Jag kommer oftast att skriva ut var du kommer att behöva skriva koden med radnummer eller liknande. Detta för att göra det lättare för dig att följa med. Jag kommer också att skriva ut hela koden i slutet av varje exempel. Därför är det viktigt för dig att veta hur en kommentar i html fungerar, kommentarerna kommer jag använda för att markera saker. En kommentar i html skrivs såhär: 

<!-- Allt mellan dessa streck är en kommentar -->

Allt inom <!-- och --> kommer inte att visas i din webbläsare. Det är en kommentar som bara du som utvecklare kan se. Det är ett bra sätt att markera saker som du vill komma ihåg eller förklara vad som händer i koden. Det är också ett bra sätt att "kommentera bort" kod som du inte vill ska köras. Detta är väldigt användbart när du ska testa saker.

Du kan välja att ta bort mina kommentarer eller att ha dem kvar. Jag kommer att utgå ifrån att de är borttagna i mina exempel.

Hello World

#

För att göra vår första webbsida så skriver vi in följande kod mellan body-taggarna: 

<!-- På rad nio i strukturen skriver vi detta: -->
<h1>Hello World</h1>

h1 - h6 är rubriker (header *1-6). Det finns 6 olika rubriker som vi kan använda oss av. h1 är den största rubriken och h6 är den minsta rubriken. Sökmotorer använder sig av rubriker för att indexera sidan. Det är därför viktigt att vi använder oss av rubriker på ett korrekt sätt om vi vill att vår sida ska kunna indexeras på till exempel Google.

Nu ska vi spara vår fil och starta vår liveserver. Tryck på spara och sedan på Go Live i Visual Studio Code (längst ner till höger). Din webbläsare kommer att öppnas och du kommer att se din webbsida. Om du inte ser din webbsida så kan du manuellt öppna din webbläsare och skriva in adressen 127.0.0.1:5500 utan https eller http. Detta är din lokala ip-adress och du kommer att se din webbsida där. Du kommer att se en rubrik som säger Hello World.

Vår första webbsida är skapad 🥳
